# ГОСТ 34.12-2015  
## Блочный шифр «Кузнечик» (Grasshopper)

---

# 1. Общие сведения

**ГОСТ 34.12-2015** — российский стандарт, определяющий современные симметричные блочные шифры.  
Стандарт введён в действие в 2016 году.

Документ описывает два алгоритма:
- «Магма» (64-битный блок)
- «Кузнечик» (128-битный блок)

Данный файл полностью посвящён алгоритму **«Кузнечик»**.

Назначение:
- Шифрование данных
- Использование в СКЗИ
- Защищённые каналы связи
- VPN
- Государственные криптосистемы РФ

---

| Параметр | Значение |
|-----------|----------|
| Размер блока | 128 бит |

# 3. Общая схема алгоритма

Шифр представляет собой подстановочно-перестановочную сеть (SPN).

Раундовая функция (1–9 раунд):


X = L(S(X ⊕ Ki))


Финальный (10-й) раунд:


X = X ⊕ K10


Где:

- ⊕ — побитовое сложение по модулю 2 (XOR)
- Ki — раундовый ключ
- S — нелинейное преобразование (S-блок)
- L — линейное преобразование

---

# 4. Раундовые преобразования

---

## 4.1 Операция XOR


C = A ⊕ B


Используется:
- Перед S-преобразованием
- В финальном раунде

---

## 4.2 S-преобразование

Каждый байт 128-битного блока заменяется по фиксированной таблице из 256 значений.


Si = S[Xi]


Начало таблицы S-box:


FC EE DD 11 CF 6E 31 16
FB C4 FA DA 23 C5 04 4D
E9 77 F0 DB 93 2E 99 BA
17 36 F1 BB 14 CD 5F C1
...


(Полная таблица содержится в официальном тексте стандарта)

---

## 4.3 Линейное преобразование L

Преобразование L = 16 последовательных применений функции R.

### R-преобразование:


R(a15, a14, ..., a0) =
(a0 ⊕ (c0·a0 ⊕ c1·a1 ⊕ ... ⊕ c15·a15), a15, ..., a1)


Умножение выполняется в поле:


GF(2^8)
с неприводимым полиномом:

x^8 + x^7 + x^6 + x + 1


Вектор коэффициентов C:


C = (148, 32, 133, 16,
194, 192, 1, 251,
1, 192, 194, 16,
133, 32, 148, 1)


---

# 5. Генерация раундовых ключей

---

## 5.1 Исходный ключ

256-битный ключ делится на две части:


K = (K1, K2)


Каждая часть по 128 бит.

---

## 5.2 Константы Ci

Генерируется 32 константы:


Ci = L(Vi)

Где:
- Vi — 128-битный вектор с номером i
- i ∈ [1..32]

---

## 5.3 Функция F

F(K1, K2, Ci):
T = L(S(K1 ⊕ Ci))
return (T ⊕ K2, K1)

После 32 итераций получается 10 раундовых ключей:

K1, K2, ..., K10

---

# 6. Полный алгоритм шифрования

X = Plaintext

for i = 1..9:
X = L(S(X ⊕ Ki))

Ciphertext = X ⊕ K10

---

# 7. Расшифрование

Используются обратные преобразования:

- Обратный S-блок
- Обратное L-преобразование
- Те же ключи в обратном порядке

# 8. Тестовый вектор (обязательная проверка)

# 9. Критически важные моменты реализации

---

## 1. Арифметика GF(2^8)

- Полином: x^8 + x^7 + x^6 + x + 1
- Нельзя использовать AES-полином
- Любая ошибка → полностью неверный результат

---

## 2. L-преобразование

- Это 16 последовательных R-операций
- Чаще всего реализуется через предвычисленные таблицы
- Возможна оптимизация через T-tables

---

## 3. Порядок байтов (Endian)

- Используется массив из 16 байтов
- Ошибки в порядке байтов — самая частая проблема

---

## 4. Генерация ключей

- 32 константы
- 32 итерации F
- Получение 10 ключей
- Строгий порядок применения

---

## 5. Проверка по тестовым векторам

Реализация считается корректной только после прохождения официального теста.
